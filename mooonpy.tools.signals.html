mooonpy.tools.signals.data_extension(xdata: ndarray | list, ydata: ndarray | list, lo: Number = 1, hi: Number = 1) → Tuple[ndarray, ndarray, int, int][source]ÔÉÅ

Function to extend data at the ‚Äúlo‚Äù and ‚Äúhi‚Äù end. This function assumes xdata is increasing and ydata is the dependent variable. This function replaces the usage of scipy‚Äôs ‚Äúsignal.filtfilt‚Äù or ‚Äúsignal.sosfiltfilt‚Äù padtype=<‚Äôodd‚Äô or ‚Äòeven‚Äô or ‚Äòconst‚Äô or None>, with greater functionality.

For example the padtype=<‚Äôodd‚Äô or ‚Äòeven‚Äô or ‚Äòconst‚Äô or None> implementation assumes that the padding on the ‚Äúlo X-end‚Äù will be the same as the padding that should be used on the ‚Äúhi X-end‚Äù, however for stress-strain data, this is usually not the case.

lo/hi integer meanings can be thought of as quadrant numbers for ‚Äúlo‚Äù and have the same meaning for ‚Äúhi‚Äù, but at the tail end of the data. For example:

  • 1 = means shut off data mirroring

  • 2 = means mirror data (lo=mirrored into quadrant 2)

  • 3 = means apply 180-degree rotation (lo=mirrored into quadrant 3)

  • 4 = means flip the Y-data and append at the end of the Y-data

Parameters:

  • xdata (Array1D) ‚Äì X data

  • ydata (Array1D) ‚Äì Y data

  • lo (Number) ‚Äì Left side quadrant setting, defaults to 1.

  • hi (Number) ‚Äì Right side quadrant setting, defaults to 1.

Returns:

Extended data as X / Y, and lo_trim / hi_trim slicing index to access original data within the extension.

Tuple[np.ndarray, np.ndarray, int, int]
